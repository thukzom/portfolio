Melanoma Detection Using Deep Learning

Python · Pandas · Scikit-learn · PyTorch · ResNet18 · OpenCV · Gradio

Team capstone (with Maria Vasai, Selam Saleh, Olivia Sampson) built on the SIIM-ISIC dermoscopic dataset, where malignant cases made up less than 2% of the data — a roughly 50:1 imbalance that biased naive models toward always predicting "benign," the single most dangerous failure mode for a screening tool.

Approach

  • Extracted deep visual features with a pre-trained ResNet18 and combined them with patient metadata (age, sex, anatomical site) after handling missing values and encoding categorical fields.
  • Reduced dimensionality through feature selection, then applied Random Over Sampling to counter the class imbalance before model training.
  • Evaluated multiple classifiers and selected Logistic Regression as the strongest performer, then tuned its decision threshold specifically to prioritize recall — minimizing missed malignant cases even at the cost of more false positives.
  • Shipped a Gradio web app where a user uploads a lesion image with patient details and gets back a benign/malignant prediction with class probabilities.

Key Results

33,000+ images analyzed Recall: 0.78 ROC-AUC: 0.846 ~50:1 class imbalance handled

Business / Clinical Impact

Deliberately optimized for recall over precision, because in a screening context a missed malignant case is far more costly than a false alarm that a dermatologist quickly rules out. Deployed as a working demo, the tool models how a triage layer could flag high-risk lesions for priority review — cutting the images a clinician has to look at first, without adding meaningful risk of a missed cancer slipping through.

Retail Customer Spending Analysis

Python · Pandas · Matplotlib · Scikit-learn

Segmented 300,000+ retail records to test a simple, common assumption in retail marketing — that spending is mostly explained by demographics — and found the data disagreed.

Approach

  • Applied regression, classification, and K-means clustering to compare how well transactional behavior versus static demographics predicted total spending.
  • Segmented customers into 8 behavioral clusters ranging from roughly 17,000 to 78,000 customers each, revealing distinct groups that a single "average customer" view would hide.
  • Tested spending distributions across four age bands (18–25, 26–35, 36–50, 51–70) and found medians moved by less than $100 across the entire range, with heavily overlapping spreads in every group.
  • Built dashboards translating this into a concrete recommendation for the marketing team.

Key Results

300,000+ records analyzed 8 behavioral clusters Age explained <$100 spend variance Largest cluster: ~78K customers

Business Impact

Age-based segmentation — a common default for retail targeting — turned out to be a weak lever here: an 18-year-old and a 65-year-old showed nearly identical spending distributions. The behavioral clusters, by contrast, isolated a ~78,000-customer high-frequency segment worth prioritizing directly. The recommendation: shift campaign targeting and budget from age brackets to purchase-frequency segments, which should improve conversion per marketing dollar without needing new data collection.
